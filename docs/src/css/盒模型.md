## 说说对盒模型的理解
在HTML页面中的所有元素都可以看成是一个盒子

盒子的组成：内容content、内边距padding、边框border、外边距margin

盒模型类型：标准盒模型、IE盒模型（怪异盒模型）

通过box-sizing修改盒模型: border-box(IE盒模型) | content-box(默认值, 标准盒模型) | inherit

**标准盒模型**
- 盒子高度/宽度 = content(width/height) + padding + border + margin
- width / height = content
- box-sizing: content-box(default)
- padding 和 border 会增加实际占用空间
- 适合内容大小已知但需要通过 padding 和 border 再做装饰的情形，比如文字块、图片容器等

**怪异盒模型**
- 盒子高度/宽度 = width/height + margin
- width / height = content + padding + border
- box-sizing: border-box
- IE 8 及以上完全支持，IE 7 及以下不支持；Chrome 10+、Firefox 29+、Safari 5.1+、Edge、Opera 等全部原生支持；iOS Safari、Android 浏览器、Chrome/Firefox for Android 等都支持
- 现代开发推荐使用，避免复杂计算
- 适合固定尺寸的布局模块或卡片组件，便于控制总宽高不被 padding/border 影响；常与栅格、响应式布局、可重用组件配合使用，以及现代框架和 UI 库（常全局设置 border-box 以简化布局计算）。
