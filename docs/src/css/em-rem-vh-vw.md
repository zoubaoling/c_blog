## em / rem / vh / vw / px 速记

> 常见考点：单位基准、适用场景、响应式策略。

### 单位快览
- **px（绝对单位）**：设备像素，固定值。布局精准、跨环境不缩放，常用于边框、阴影等细节。
- **em（相对父级字体）**：1em 等于当前元素的 `font-size`。有级联效应，嵌套修改会层层累积，适合局部缩放（如内边距随字体变化）。
- **rem（相对根元素）**：始终基于 `html` 的 `font-size`，不受嵌套影响，是响应式排版的主力单位。
- **vh / vw（视口单位）**：`1vh` 为视口高度的 1%，`1vw` 为宽度的 1%。用于全屏版式、背景、横幅等占满视窗的场景。移动端需注意软键盘弹出导致高度变化。
- **%（相对父容器）**：尺寸/位置受父元素影响，定位元素有特例：`absolute` 参考最近定位祖先，`fixed` 相对视口。
- **vmin / vmax**：取视口宽高中较小/较大的百分比，适合需要保持比例的组件。

### 常用策略
- **基准设置**：`html { font-size: 62.5%; }` → 1rem ≈ 10px，方便 rem 与设计稿换算。
- **组合用法**：框架骨架用 rem，自适应背景或弹窗用 `vh/vw`，微调依旧用 px。
- **兼容注意**：安卓低版本 WebView 对 `vw/vh` 支持不一；`vh` 在移动端受地址栏收缩影响，必要时改用 JS 计算或 `min-height`。
  - 软键盘弹出时 `100vh` 会随可视高度缩小，常见 UI 抖动，可用：
    - `height: 100dvh`（现代浏览器的动态视口单位，会随键盘变化实时更新）；
    - JS 监听 `resize`，将 `window.innerHeight` 转成 `--vh` 变量，使用 `height: calc(var(--vh) * 100);`；
    - 关键内容设置 `min-height: 100vh`，多余部分允许滚动，避免硬撑满高。

### 面试速答模板
1. 先讲“基准”：px 固定，em 看父级，rem 看根节点，vh/vw 看视口。
2. 再讲“场景”：精准布局用 px，字体/内边距用 em/rem，整屏背景/弹窗用 vh/vw。
3. 最后提“坑位”：em 累积、vh 在移动端不稳定、rem 需配合根字体设置。
