## 响应式设计面试速记

> 核心定义：同一套页面在不同设备/视口下自动调整布局、字体与媒体尺寸，带来一致的体验。

### 一句话回答结构
1. **理念**：内容优先，设计对不同屏幕做自适应，而不是为每个终端做独立页面。
2. **三大支柱**：流式布局 + 弹性媒体 + 媒体查询。
3. **实现流程**：移动端起步（Mobile First）→ 设置 `viewport` → 用 Flex/Grid 构建 → 媒体查询调优。

---

### 核心原理拆解
- **流式布局**：使用百分比、`vw/vh`、`rem` 等相对单位，让块随视口伸缩。
- **弹性媒体**：图片/视频 `max-width: 100%; height: auto;`，或 `object-fit: cover;`。
- **媒体查询**：按断点加载不同样式，常见断点：`<768px` 手机、`768~1024px` 平板、`>1024px` 桌面。写法示例：
  ```css
  @media (max-width: 768px) {
    .layout { flex-direction: column; }
  }
  ```
- **移动优先**：默认样式服务小屏，通过 `min-width` 断点逐级“增强”。
- **视口设置**：`<meta name="viewport" content="width=device-width, initial-scale=1">`，确保 CSS 像素与设备宽度一致。

---

### 常用实现套路
- **布局策略**
  - 优先 Flex/Grid；辅助使用百分比宽度或 `calc()`。
  - 老项目可保留浮动或多栏布局，但需注意清除浮动。
- **字体与间距**
  - `rem` 与根字体结合媒体查询（或 JS）动态调整。
  - `vw`/`clamp()` 创建自适应排版：`font-size: clamp(1rem, 2vw, 1.5rem);`
- **组件与导航**
  - 小屏隐藏侧栏、切换为抽屉/汉堡菜单。
  - 使用 `display: none`, `visibility`, 或 `transform` 切换模块，可强调动画性能。
- **响应式图片**
  - `<img srcset>` 或 `<picture>` 提供不同分辨率资源；配合懒加载提升性能。

---

### 单位速览
- `%`：相对于父元素尺寸；注意不同属性的参考基准可能不同。
- `rem`/`em`：相对于根/父元素字体，可联动字体缩放。
- `vw/vh`：相对于视口宽高的 1%；用于全屏容器或流式组件。

---

### 典型问答亮点
- **如何选断点？** 根据设计稿或数据统计的设备宽度，不必照搬“Bootstrap 断点”。
- **为什么强调 Mobile First？** 小屏约束更严格，先解决核心内容，再向大屏扩展；也能减少覆盖式写法。
- **性能与可维护性**：组件化样式、CSS 变量、避免过度嵌套媒体查询；使用设计系统保持一致性。
- **局限性**：需要大量测试和调优；复杂交互可能仍需独立适配；不同设备的触控/交互差异要单独处理。
