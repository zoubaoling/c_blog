## 精灵图 vs Base64

### 核心概念
- `精灵图 (CSS Sprite)`：将多张小图合并成一张位图，通过 `background-position` 精确定位，减少 HTTP 请求。
- `Base64`：把二进制图片编码成文本，内嵌在 HTML/CSS 中，绕过网络请求。

### 优势对比
- `精灵图`
  - 共享一次下载，适合聚合大量小图标。
  - 浏览器可单独缓存整张 sprite，后续访问更快。
- `Base64`
  - 无跨域、无需额外请求，适合首屏小图。
  - 文本形式易于通过内联样式或 JS 动态注入。

### 劣势对比
- `精灵图`
  - 维护成本高：任意改图都要重新切图、定位。
  - 初次加载体积大；响应式、Retina 需多套 sprite。
- `Base64`
  - 编码后体积约增 33%，会拖慢 CSS/HTML 下载。
  - 无独立缓存，每次页面解析都要重新解码。

### 典型场景
- `精灵图`：一组稳定的小图标（导航、按钮状态）且需缓存。
- `Base64`：偶发的小装饰（<10 KB）、首屏关键资源、邮件模板。

### 面试速记
1. 先说“本质”：精灵图靠合并请求，Base64 靠编码内联。
2. 再说“取舍”：精灵图更利缓存，Base64 更快首屏。
3. 最后补“限制”：精灵图难维护，Base64 体积大无缓存。
